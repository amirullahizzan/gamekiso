提出場所：teams->ゲームプログラム基礎->授業（本日）->ファイル->提出

プロジェクトフォルダの名前を【 プロジェクト名_番号_自分の名前 】のように変更して提出してください。

以下の変更を実装することで【 こういう感じで.exe 】のような挙動になるようにしてください。


課題１：状況に合わせたＳＥの再生

	今回の課題用のサウンドデータはすでに読み込んでいます。
	↓
	se_walk = LoadSoundMem( "data/se_walk.mp3" );	//	足音
	se_door = LoadSoundMem( "data/se_door.mp3" );	//	ドアを開ける音
	se_rain = LoadSoundMem( "data/se_rain.mp3" );	//	雨の音

	これらのＳＥデータをゲームの状況に合わせて再生を行います。

	・プレイヤーが移動しているとき　→　足音

	・場面が切り替わるとき　　　　　→　ドアを開ける音

	・フィールドエリアにいるとき　　→　雨の音

	さらに、よりその場面になっている感を出すために背景も描画して切り替えます。
	それぞれの場面が分かるような背景画像も用意しています。
	image_room.bmp　　　←　室内っぽいフローリングの画像
	image_field.bmp 　　←　外っぽい画像
	（今はどの場面かを見るような変数が必要です）


課題２：雨の描画表現

	課題１で、プレイヤーが外にいるときに雨の音を再生するようにしましたが、
	「こういう感じで.exe」ではさらにその状況が分かるように雨と分かるような表現もしています。

	以下のような処理をしています
	・雨１粒用の座標変数を複数用意
	・画面上のどこかランダムに座標を設定
	・画面下まで行ったら上にを繰り返す
	・その座標から線を引きます
		線描画関数：DrawLine( x1, y1, x2, y2, GetColor( 255, 255, 255 ) );

	線を描画するためには２点の座標が必要ですが、
	１点目はそのランダムな座標、２点目はそこより少しだけ上の座標にしています。

	この雨の描画をプレイヤーが外にいる状態の時だけ描画します。


ちなみに
	画面サイズ　　　　　　８００ｘ４５０
	背景画像サイズ　　　　８００ｘ４５０
です。


各処理に関して、既に宣言してある変数や関数だけでは処理が実装できない場合、
各自で変数や関数を追加した上で実装をおこなってください。
座標や色・スピードなど細かい数値の指定は特にないので、各自でいい感じの数値を設定することで
【 こういう感じで.exe 】のような動きになるように実装してください。
